You are an expert Flutter developer.
Create a user profile screen in Flutter that replicates the design and functionality of the provided PHP web page (`user-view-profile.php`).

### **Design Requirements**
The screen should have a modern, clean UI similar to the web version:
1.  **Header:**
    *   Title: "My Profile"
    *   User info (Name, Role) and a profile icon.
2.  **Profile Card:**
    *   Gradient background (Teal/Green theme).
    *   Circular avatar icon.
    *   User Name and Role (e.g., Employee).
    *   "Joined Date" badge.
3.  **Statistics Row:**
    *   Three cards displaying: "Total Trips", "Approved", and "Pending" counts.
4.  **Info Grid:**
    *   Cards for Email, Phone, and Address with corresponding icons.
5.  **Action Buttons:**
    *   "Update Profile" (Primary color).
    *   "Logout" (Red/Danger color).

### **API Integration Logic**
To populate this screen, you need to fetch data from the backend.

**1. API Endpoint:**
*   **Endpoint:** `GET /user/profile.php`

**Recommended API Structure:**
*   **Request:** `GET /api/user/profile.php`
*   **Headers:** `Authorization: Bearer <token>`
*   **Response:**
    ```json
    {
      "success": true,
      "data": {
        "user": {
          "first_name": "John",
          "last_name": "Doe",
          "email": "john@example.com",
          "phone": "1234567890",
          "address": "123 Main St",
          "role": "EMPLOYEE",
          "created_at": "2023-01-15"
        },
        "stats": {
          "total_bookings": 12,
          "approved_bookings": 8,
          "pending_bookings": 2
        }
      }
    }
    ```

**2. Flutter Implementation Details:**
*   **State Management:** Use `Provider` or `Riverpod` (or `setState` for simplicity) to handle loading, success, and error states.
*   **Network Calls:** Use `Retrofit` or `http` package.
*   **UI Components:**
    *   Use `Card` widgets with `elevation` for the stats and info blocks.
    *   Use `Container` with `BoxDecoration` for the gradient header.
    *   Use `Column`, `Row`, and `GridView` (or `Wrap`) for layout.

**3. Logout Logic:**
*   Call the `POST /auth/logout.php` endpoint.
*   Clear local storage (tokens).
*   Navigate back to the Login screen.

**Task:**
Write the complete Flutter code for the `UserProfileScreen` widget and the necessary data models. Assume a `UserRepository` exists for fetching data.
